Sistema de Gestión de Biblioteca

Proyecto de Investigación Nro. 2 — Base de datos No Relacional MongoDB

Acerca del proyecto

Sistema de información para la gestión de una biblioteca, desarrollado con Python, Flask y pymongo, conectado a MongoDB Atlas. Permite administrar autores, libros, ediciones, copias, usuarios y préstamos mediante operaciones CRUD completas, además de consultas con aggregation pipelines.

Reglas de creación

  • Autor: al crearlo se debe asociar al menos un libro. Si el libro no existe, el sistema pregunta si desea crearlo.
  • Libro: se crea junto con su edición y primera copia. Permite asociar múltiples autores.
  • Edición: el libro debe existir previamente. Se crea junto con su primera copia.
  • Copia: la edición (ISBN) debe existir previamente.
  • Préstamo: la copia y el usuario deben existir previamente.

Reglas de eliminación

  • Autor: elimina sus parejas en autorea. Los libros que queden sin autores se eliminan en cascada.
  • Libro: elimina sus ediciones y copias. Los autores que queden sin libros se eliminan.
  • Edición: elimina sus copias. Si el libro queda sin ediciones, se elimina en cascada.
  • Copia: solo se elimina la copia. La edición se mantiene aunque quede sin copias.
  • Préstamos: nunca se eliminan en cascada, son historial del negocio.

Reglas de actualización

  • Actualizar el nombre de un autor se propaga a la colección autorea.
  • Actualizar el título de un libro se propaga a autorea y edición.
  • Actualizar el ISBN de una edición se propaga a copia y préstamo.
  • Actualizar el número de una copia se propaga a préstamo.
  • Actualizar año o idioma de una edición no requiere cascada.

Cadena de dependencias

autor ← autorea → libro ← edicion ← copia ← prestamo → usuario
